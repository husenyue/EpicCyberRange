## 单靶标元数据标注步骤

数据结构（json）

```json
{
  "name": "weblogic-cve-2020-14882",
  "image": "vulhub/weblogic:12.2.1.3",
  "contracts": {
    "ingress": [7001],
    "egress": ["ldap_server"],
    "vuln_type": "rce",
    "poc_script": "pocs/weblogic/cve_2020_14882.py"
  }
}
```

### 收集name字段

- 直接输入靶标名称

- 输入关键词，在[Vulhub 官方仓库](https://github.com/vulhub/vulhub)中中检索找到对应靶标，输出完整靶标名称`name`

### 收集image、contracts.ingress字段

- 拆分靶标名称

- 进入该靶标的目录，查看`docker-compose.yml`文件，获取`image`、`contracts.ingress`字段

提取文件中指定的镜像地址，格式为`"image": "vulhub/组件名:版本"`，需确认镜像版本与漏洞环境匹配。

在上述docker-compose.yml文件中，查找ports配置项（如7001:7001）；

提取宿主机映射的端口号（仅保留数字），整理为数字数组格式，如`"ingress": [7001]`；

确认端口是该漏洞环境对外提供服务的核心端口（如 Weblogic 默认管理端口 7001）。

### 收集contracts.egress字段（依赖的外部服务）

- LLM查阅该漏洞的技术文档/EXP说明，确认漏洞利用时依赖的外部服务；

- 将依赖服务名称整理为字符串数组，格式为`["服务名_server"]`（如["ldap_server"]）；

- 若无明确依赖服务，填写["unknown"]。

### 收集contracts.vuln_type字段（漏洞类型）

- LLM通过 CVE 编号查询漏洞详情（如 CVE-2020-14882 为远程代码执行漏洞）；

将漏洞类型标准化为小写缩写（rce/远程代码执行、sqli/SQL注入、xss/跨站脚本、lfi/文件包含等），仅保留核心缩写（如rce）。
